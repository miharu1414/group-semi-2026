# 要件定義

## 目的

研究室メンバーがゼミ予定と担当者を一目で確認・更新できるWebカレンダーを提供する。

## ユーザー

- 研究室メンバー
- 教員
- スマートフォンまたはPCから利用するユーザー

## MVP機能

| ID | 機能 | 内容 |
| --- | --- | --- |
| F-01 | 月次カレンダー表示 | 現在月の予定をグリッド表示 |
| F-02 | 予定表示 | 日付、ゼミ種別、担当者A/B/C、メモを表示 |
| F-03 | 予定追加 | 日付クリックから予定を作成 |
| F-04 | 予定編集・削除 | 既存予定を更新または削除 |
| F-05 | メンバー管理 | メンバーの追加・編集・削除 |
| F-06 | 担当者選択 | 登録済みメンバーから担当者を選択 |
| F-07 | 月移動 | 前月・次月・今日へ移動 |

## 非機能要件

| ID | 要件 | 内容 |
| --- | --- | --- |
| NF-01 | 表示速度 | 初期表示を軽く保つ |
| NF-02 | レスポンシブ | スマートフォンとPCで利用できる |
| NF-03 | アクセシビリティ | ラベル、ARIA、キーボード操作を考慮する |
| NF-04 | 秘密情報管理 | Firebase認証情報をリポジトリに含めない |

## スコープ外

- ユーザー認証・権限管理
- カレンダーのエクスポート
- Slack通知
- 繰り返し予定の自動生成
