# 要件定義書

> **整合性**: このファイルを変更した場合、`INSTRUCTIONS.md` の整合性チェックリストを実行してください。

## 目的

研究室メンバーが **ゼミの予定と担当者を一目で把握できる** Webカレンダーアプリを提供する。  
Excelや口頭連絡による管理のコストを削減し、担当者の見落としを防ぐ。

---

## ユーザー

- 研究室メンバー（学部生B3〜B4、修士M1〜M2、博士D1〜D3、教員）
- 認証不要（研究室内ネットワーク利用を前提）
- スマートフォン・PCどちらでも使用する

---

## 機能要件

### 必須機能（MVP）

| ID | 機能 | 詳細 |
|---|---|---|
| F-01 | 月次カレンダー表示 | 現在月のカレンダーをグリッドで表示 |
| F-02 | ゼミ予定の可視化 | 各日にゼミ種別（輪読/全体/研究）と担当者A/B/Cを表示 |
| F-03 | ゼミ予定の追加 | 日付クリックでモーダルを開き登録 |
| F-04 | ゼミ予定の編集・削除 | 予定クリックで編集、削除は確認ダイアログあり |
| F-05 | メンバー管理 | 研究室メンバーの追加・編集・削除、役職（学年）設定 |
| F-06 | 担当者ドロップダウン | メンバー登録済みの場合はセレクトBOXで選択 |
| F-07 | 月移動 | 前月・次月・今日ボタンで月を切り替え |

### 非機能要件

| ID | 要件 | 目標値 |
|---|---|---|
| NF-01 | 初期表示速度 | 3秒以内 |
| NF-02 | モバイル対応 | iPhone SE（375px）〜デスクトップで正常動作 |
| NF-03 | アクセシビリティ | ARIA属性、キーボード操作対応 |
| NF-04 | 可用性 | Cloudflare Pages SLAに準拠 |
