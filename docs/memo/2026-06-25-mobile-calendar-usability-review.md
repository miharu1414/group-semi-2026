# 2026-06-25 mobile-calendar-usability-review

## 実施内容

- スマホ幅の月表示で、予定を下端の細い色バーではなく日付セル内のコンパクトな予定チップとして表示するようにした。
- スマホでは各日付セルを少し高くし、最大2件の予定と追加件数を読み取れるようにした。
- 月ヘッダーの余白と文字サイズを調整し、狭い画面で月名・ナビゲーション・今日ボタンが詰まりにくいようにした。
- 予定一覧の並び順を日付、開始時刻、タイトルで統一した。
- 予定編集で別月へ移動した場合に、現在月のローカル表示から残らないようにした。
- `activity_id` をフォームデータにも含め、活動設定に基づく色・ラベル拡張の設計と保存データを揃えた。

## 背景

PC表示では予定カードが十分に読める一方、スマホ表示では予定が日付セルの下端に寄った細いバーのみになり、内容を判別しづらかった。月全体の一覧性は保ちつつ、指で日付をタップして詳細を見る導線を維持するため、セル内のコンパクト表示へ変更した。

## 確認項目

- `npm run lint`
- `npm run docs:check`
- `git diff --check`

`npm run build` と `npm run dev` は、このローカル環境の Node.js v24.18.0 で Next.js 14.2.5 の Windows SWC バイナリが `not a valid Win32 application` となり起動前に失敗した。コード差分起因のエラーではなく、SWC ネイティブバイナリの実行環境問題として扱う。
